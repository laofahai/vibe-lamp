# Vibe Lamp Core V1 设计评审（schematic ↔ BOM ↔ spec 交叉核对）

本文件对照 `v1_spec.md`、`v1_schematic.md`、`v1_bom.csv` 三份输入，逐项核对网络/器件是否齐全、是否存在矛盾或欠定义项，并给出可执行结论。
评审目标：在进 KiCad 画板之前，把"画到一半才发现缺料/缺网络/选型没定"的坑提前清掉。

适用范围：28×28mm 圆角方板、2 层、ESP32-C3-MINI-1、工厂贴片、用户不焊接。

---

## 0. 结论摘要（先看这里）

总体方案自洽，可以进入 KiCad。但下列 **8 项必须在画原理图/下单前定稿**，否则会卡在封装指派或贴片环节：

| 编号 | 风险项 | 严重度 | 处置 |
|---|---|---|---|
| R-01 | LDO 具体型号未定（影响封装 + C1/C2 容值） | 高 | 选定型号，见 §2 |
| R-02 | RGB LED 封装/共阴脚序未定（封装无法指派） | 高 | 选定型号并按其脚序建符号，见 §4 |
| R-03 | USB-C 16 脚 vs 24 脚未定（影响封装与连线） | 中 | 定 16 脚，见 §3 |
| R-04 | 原理图提到 ESD 器件位，BOM 无对应料 | 中 | 加 DNP 位 D2，见 §3 |
| R-05 | 原理图提到 VBUS 保险丝/0Ω 跳线位，BOM 无对应料 | 中 | 加 F1（0Ω/DNP），见 §3 |
| R-06 | 固件 `c3_rgb` env 未带 `-DPIN_RESET_BTN=10`，按钮默认接 GPIO0 | 高 | 改固件，见 §6 |
| R-07 | 330Ω 三路同值会导致蓝/绿偏暗、红偏亮（色彩不平衡） | 中 | 见 §4，决定按通道调阻值还是靠固件校色 |
| R-08 | BOM 未标清哪些是 DNP（不贴件），CPL 易把空焊盘当贴片 | 中 | 见 §5，统一标 DNP |

下面是逐节明细。

## 1. 电源/网络完整性核对

主网络清单（三份文档合并后应存在且唯一）：

- `+5V`：USB-C VBUS → LDO IN，以及 WS2812 预留焊盘 5V。
- `+3V3`：LDO OUT → 模组 3V3、EN 上拉、各去耦电容、TP1。
- `GND`：公共地。
- `USB_DP` / `USB_DM`：USB-C D+/D- ↔ 模组原生 USB（C3：D+ = GPIO19，D- = GPIO18，见 §3）。
- 信号网络：`RGB_R`/`RGB_G`/`RGB_B`、`WS2812_DIN`、`BTN_RESET`、`EN`、`U0RX`/`U0TX`、`IO9`。

核对结果：spec / schematic 描述一致，无缺失主网络。**但** 以下几处需在原理图里显式补全（文档里是文字描述，画图时容易漏）：

1. **USB-C VBUS 是 4 个 pin、GND 是 4 个 pin**（16 脚连接器，见 §3），必须全部并接到 `+5V`/`GND`，不能只接一个。
2. **USB-C D+/D- 各有两组**（A 面 + B 面，A6/B6、A7/B7）。要做"正反插都能用"，必须把两个 D+ 短接成 `USB_DP`、两个 D- 短接成 `USB_DM`。文档只写了"D+/D-"，画图时别漏掉短接。
3. **GPIO9（BOOT）默认上拉**：C3 内部有上拉，但产品板建议外部不要把 IO9 拉低；TP6 仅作"下载时手动短到 GND"用。原理图里 IO9 走线到 TP6 即可，不要放下拉。

## 2. LDO（U2）— R-01 高风险，必须定型

现状：BOM 写"输出 ≥500mA，SOT-23-5/SOT-223/DFN，不优先 AMS1117"，schematic 写"立创可 SMT、≥500mA、输入输出按手册放电容"。**型号未定，导致封装无法选、C1/C2 容值无法定。**

技术核算：
- 负载只有一颗 ESP32-C3-MINI-1 + 一颗 RGB LED。C3 射频发射瞬时峰值电流约 350mA 量级，平均远低。**500mA LDO 是临界够用，建议留余量选 500–600mA。**
- 输入 5V、输出 3.3V，压差 1.7V，对任何 LDO 都富余，不挑低压差型号。
- 关键是 **瞬态响应 + 输入大电容**，否则射频峰值会拉低 3V3 导致复位/掉线。

推荐定型（任选其一，都是立创/JLCPCB 常备、陶瓷电容稳定型）：
- **AP2112K-3.3TRG1**，SOT-23-5，600mA，手册要求 1µF 入 / 1µF 出（陶瓷）。
- **ME6211C33M5G-N**，SOT-23-5，500mA，1µF 入 / 1µF 出。
- **RT9013-33GB**，SOT-23-5，500mA，1µF 入 / 1µF 出。

定型后联动修改：
- C1/C2 由"1–10µF 待定"**固定为 1µF 0603**（按上述手册），并在靠 LDO 处再加一个 **10µF（可用 C3 主储能那颗兼顾，或单独加 10µF 输入电容）**。建议输入端 10µF + 1µF 并联，吸收 USB 线缆电感引起的浪涌。
- 封装统一为 `Package_TO_SOT_SMD:SOT-23-5`。

> 不选 AMS1117 的理由（与 spec 一致）：AMS1117 静态电流大（5–10mA）、压差高、SOT-223 占地大、发热明显，对 28×28 小板和"插 USB 长期待机"不划算。

## 3. USB-C（J1）+ USB 数据 — R-03 / R-04 / R-05

### R-03 连接器脚数定型
现状 BOM："16 脚或 24 脚"。本板只用 USB2.0 供电 + 原生 USB 数据，**不需要 SBU、不需要全功能 Type-C**。

结论：**定 16 脚电源型 Type-C 母座**（带 VBUS×4 / GND×4 / CC1 / CC2 / D+×2 / D-×2）。推荐 HRO `TYPE-C-31-M-12`（立创 C165948），KiCad 标准封装 `Connector_USB:USB_C_Receptacle_HRO_TYPE-C-31-M-12` 直接可用，省去导入。

### CC 电阻核对
schematic：CC1→5.1k→GND、CC2→5.1k→GND（即 R6、R7）。**正确**——本设备是纯受电端（UFP/sink），两路 CC 各下拉 5.1k 表示 Rd，使源端按默认 5V/最高电流供电。无矛盾。

### USB 数据脚号补全
文档只写"ESP32-C3 原生 USB D+/D-"，未写具体 GPIO。**画图时必须落实：ESP32-C3 原生 USB → GPIO18 = D-（`USB_DM`），GPIO19 = D+（`USB_DP`）。** 在 build guide 的网表里已写死。

### R-04 ESD 器件位（BOM 缺料）
schematic 写"USB 口附近预留 ESD 器件位，V1 可按成本决定是否装"，但 **BOM 没有这一项**。这是 schematic ↔ BOM 的不一致。
结论：在原理图/BOM 里补一个 **D2 = USBLC6-2SC6（SOT-23-6）**，D+/D-/VBUS 三路 ESD，**默认标 DNP**（不贴，留位）。封装 `Package_TO_SOT_SMD:SOT-23-6`。这样"按成本决定是否装"才有物理落点。

### R-05 VBUS 保险丝/0Ω 跳线（BOM 缺料）
schematic 写"VBUS 后预留保险丝或 0Ω 跳线位"，**BOM 同样没有**。
结论：补一个 **F1**，0603/1206 位，V1 默认贴 **0Ω 跳线**（或自恢复保险丝），串在 VBUS→`+5V` 之间。这样将来要加限流/保险只换料即可。BOM 里要有这一行。

## 4. RGB LED（D1）+ 限流电阻 — R-02 / R-07

### R-02 封装/脚序未定（高风险）
现状：spec/BOM 只写"共阴 RGB 贴片灯、优先立创库存"，**没定具体型号、封装、脚序**。RGB LED 的脚序各家不同（PLCC-6 5050 / PLCC-4 / 3528 等），**封装和符号脚序必须按选定的那一颗来建**，否则会贴反、点不亮。

结论：
- 推荐 **共阴 5050 RGB LED（PLCC-6）**，立创 SMT 常备，扩散罩出光均匀。
- **选定具体 LCSC 料号后，用 `easyeda2kicad` 直接导入它的官方封装+3D**（见 build guide §footprint），不要手搓，避免脚序错。
- 原理图符号的 R/G/B 三个阳极 + 公共阴极脚序，必须与所选料号 datasheet 完全一致；丝印要标公共脚方向（spec 已要求）。

### R-07 330Ω 三路同值的色彩不平衡（中风险，真实物理问题）
固件按"高电平点亮、共阴"驱动，GPIO 经 330Ω 接 LED 阳极，阴极到 GND。按典型正向压降估算（GPIO 输出约 3.3V）：
- 红：VF≈2.0V → I≈(3.3−2.0)/330 ≈ **3.9mA**
- 绿/蓝：VF≈3.0–3.2V → I≈(3.3−3.1)/330 ≈ **0.6mA**（很弱）

后果：**蓝/绿会明显偏暗、红偏亮，白平衡偏红。** 这是真实可见的问题，不是吹毛求疵。

可选处置（二选一或叠加）：
1. **按通道分别取阻值**（硬件解决）：红用大些（470–680Ω 压暗红），蓝/绿用小些（150–220Ω 提亮）。注意单脚拉电流别超 C3 推荐值（建议 ≤20mA/脚，且多脚同亮时注意 VOH 在负载下会下降）。这会偏离当前"三路都 330Ω"的 BOM，需要更新 R1/R2/R3 阻值。
2. **保持 330Ω，靠固件 PWM 校色 + gamma**（软件解决）：当前固件有 `LEDC_RES=8`、`MAX_BRIGHTNESS=160`，可在驱动里给三通道不同增益。代价是蓝/绿本来就只有 0.6mA，PWM 只能再调暗、无法调亮，**亮度上限受限**。

建议：**优先方案 1（分阻值），因为 0.6mA 的蓝/绿亮度太低，软件救不回来。** 若坚持三路同值以简化 BOM，至少把阻值降到 ~150–220Ω 全通道（红会稍亮但可接受），别用 330Ω。这一项请与固件维护者确认后再定稿 BOM。

## 5. WS2812 预留 + DNP 标注一致性 — R-08

WS2812 相关器件：R4（DATA 串 330Ω）、C6（470µF）、P1（5V/DIN/GND 焊盘）。

现状不一致点：
- BOM 只对 **C6 标了"默认不贴"**，对 **P1 标"不贴件"**，但 **R4 没标 DNP**。
- 问题：V1 不装灯环。R4 若照贴，DATA 线（GPIO4 经 330Ω）会终止在一个开路焊盘 P1 上——电气上无害，但 **CPL/贴片文件里 R4 会被当作要贴的料**，而它驱动的负载根本不存在。

结论（统一规则，CPL 才不会出错）：
- **V1 实际要贴**：U1, J1, U2, D1, R1, R2, R3, R5, R6, R7, C1, C2, C3, C4, C5, SW1, F1(0Ω)。
- **V1 标 DNP / 不贴（留位）**：R4, C6, P1（WS2812 扩展三件），D2（ESD，按成本），TP1–TP6（测试焊盘本就无器件）。
  - 注：R4 是否贴可二选。若希望"将来焊上 P1 就能直接用"，可把 R4 照贴（成本几分钱），仅 C6/P1 留空。请在 BOM 里对每一行 **显式写 `Populate=Yes/No`**，不要靠备注文字。
- 建议在 KiCad 里用元件的 **"Do not populate" 属性 + "Exclude from BOM/Position files"**（按需）来表达，导出 CPL 时才不会把空焊盘塞进贴片坐标。

## 6. 引脚分配与固件一致性 — R-06（高风险，跨文档）

硬件用脚（来自 schematic/spec）与 C3 实际功能脚核对：

| 功能 | GPIO | C3 复用 | 是否 strapping | 核对 |
|---|---|---|---|---|
| RGB_R | 7 | 普通 IO | 否 | OK，与固件 `c3_rgb` 一致 |
| RGB_G | 6 | 普通 IO | 否 | OK |
| RGB_B | 5 | 普通 IO | 否 | OK |
| WS2812 DATA | 4 | 普通 IO | 否 | OK，与 `config.h` `PIN_WS2812=4` 一致 |
| 清网按钮 | 10 | 普通 IO | 否 | 硬件 OK，**但固件未配，见下** |
| USB D- | 18 | 原生 USB | — | 需在原理图落实 |
| USB D+ | 19 | 原生 USB | — | 需在原理图落实 |
| UART RX | 20 | U0RXD | 否 | OK（TP3） |
| UART TX | 21 | U0TXD | 否 | OK（TP4） |
| BOOT/下载 | 9 | strapping | **是** | OK，仅 TP6，默认上拉 |

未使用的 strapping 脚：GPIO2、GPIO8 也是 strapping。本设计没用到它们——**保持悬空即可，但务必不要在它们上面误布走线/下拉**（GPIO8 上电时被拉低会影响启动）。画图时确认这两脚 NC。

### R-06 固件没带 `-DPIN_RESET_BTN=10`（关键）
`firmware/include/config.h` 默认 `PIN_RESET_BTN=0`；`firmware/platformio.ini` 的 `[env:c3_rgb]` 只设了 RGB 三脚，**没有 `-DPIN_RESET_BTN=10`**。
而本硬件把清网按钮接在 **GPIO10**。

后果：**按当前固件烧录的 C3 板，清网按钮（GPIO10）不起作用**——固件监听的是 GPIO0。spec 第 71 行也只是"建议增加"，尚未落地。

结论：在 `[env:c3_rgb]` 的 `build_flags` 加上 `-DPIN_RESET_BTN=10`（与 spec 建议一致）。这是出厂固件必须改的一行，否则成品按钮是死的。请在硬件定稿的同时同步固件改动。

## 7. EN / 复位 / 去耦核对

- EN：10k 上拉到 3V3（R5）+ 可选 100nF 到 GND（C5）。与 spec/schematic 一致，**保留 C5**（上电复位更干净，几分钱）。
- 模组 3V3 去耦：10µF（C3）+ 100nF（C4）靠近模组 3V3 脚。与 spec §电源一致。**布局时务必靠近模组 3V3 焊盘，走线短。**
- 无矛盾。

---

## 8. 测试焊盘核对

TP1–TP6 = 3V3 / GND / U0RX(GPIO20) / U0TX(GPIO21) / EN / IO9。与 schematic"调试焊盘"清单逐项对应，**完整、无缺**。这些是裸焊盘/测试点，无器件，导出 CPL 时排除。

---

## 9. 机械/天线（与 PCB 约束交叉，画板前必读）

- **天线 keep-out 是本板第一约束**：ESP32-C3-MINI-1 自带 PCB 天线 + 射频匹配，**不需要任何外置天线器件**。模组天线端必须贴板边，天线正下方/正前方区域 **禁止铺铜、禁止走线、禁止过孔、禁止放器件、禁止螺丝/金属**（按 Espressif ESP32-C3-MINI-1 datasheet keep-out）。详见 build guide。
- RGB LED 在正中心，与"天线贴板边"在物理上是两个不同边，注意 28×28 内同时满足"LED 居中 + 模组天线端贴一条边 + USB-C 占一条边 + 按钮占边缘"——**布局较紧，建议先做纸面/KiCad 占位排布确认能放下**（见 build guide 布局图）。
- 若将来要外置天线/IPEX：那是 **ESP32-C3-MINI-1U** 变体（带 U.FL），属于换料 + 改封装，**不在 V1 范围**，此处仅作备选记录。

## 10. 给画板人的"定稿前必答清单"

1. LDO 选哪颗？（AP2112K-3.3 / ME6211C33 / RT9013-33，定后锁 C1/C2=1µF）
2. RGB LED 选哪颗 LCSC 料号？（定后用 easyeda2kicad 导封装，核对共阴脚序）
3. 三路限流电阻：维持 330Ω 还是按通道调（红↑ 蓝绿↓）？需与固件维护者确认色彩方案。
4. R4（WS2812 数据电阻）V1 贴还是 DNP？
5. ESD（D2）贴还是 DNP？保险丝位 F1 用 0Ω 还是自恢复保险丝？
6. 固件 `[env:c3_rgb]` 是否已加 `-DPIN_RESET_BTN=10`？（不加按钮失效）
7. USB-C 确认 16 脚电源型（C165948）。
8. 阻焊/表面处理：黑/白阻焊 + （预算允许）沉金。

以上 8 条全部有答案后，再进入 `v1_kicad_build_guide.md` 的画板流程。
